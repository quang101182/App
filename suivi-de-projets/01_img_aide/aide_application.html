🚀 Guide Interactif - Suivi de Projets & Actions 🚀

Bienvenue !

Ce guide vous aidera à prendre en main l'application de Suivi de Projets et Actions.

L'objectif principal est de centraliser et organiser le suivi des tâches, que ce soit pour différents projets, services ou même pour un usage personnel.

Naviguez à travers les étapes en utilisant les boutons Précédent et Suivant en bas de page.

Concepts Importants :

  • Configurations : Des environnements séparés (jusqu'à 10), chacun avec ses propres dossiers de sauvegarde et pièces jointes. Idéal pour gérer des contextes différents (ex: Projets Investissement, Suivi Atelier, Qualité...).
  • Profils : Au sein d'une configuration, créez des profils pour isoler les données (ex: par chef de projet, par utilisateur, par grand projet...).
  • Dossiers : Vous devez définir des dossiers locaux (ou réseau) pour stocker les Sauvegardes (.json) et les Pièces Jointes (fichiers réels).
  • Permissions : L'application nécessite votre autorisation pour accéder à ces dossiers.

Prêt ? Cliquez sur Suivant pour commencer !

Permissions et Dossiers (Crucial !)

Pour que l'application puisse sauvegarder vos données et gérer les pièces jointes, elle a besoin d'accéder à des dossiers sur votre ordinateur (ou un lecteur réseau).

  1. Allez dans l'onglet Tableau de bord.
  2. Repérez le bouton 🔒 Autoriser dossiers (Config X) (où X est le numéro de la configuration active).
  3. Si ce bouton clignote ou est en surbrillance, cela signifie que les dossiers ne sont pas définis pour la configuration active, ou que la permission d'accès doit être confirmée.
  4. Cliquez sur ce bouton. Votre navigateur vous demandera de sélectionner un dossier pour les Sauvegardes (Backup), puis un autre pour les Pièces Jointes (PJ).
  5. Vous devrez peut-être confirmer que vous autorisez le site à lire et écrire dans ces dossiers.

➡️ Cette étape est indispensable pour la sauvegarde et la gestion des fichiers. Vous devrez potentiellement le faire pour chaque Configuration que vous utilisez.

💡 Les chemins des dossiers choisis sont visibles dans l'onglet Configuration.

Configurations & Profils : Le Cœur de l'Organisation

L'application offre une flexibilité unique grâce à deux niveaux d'organisation :

1. Configurations (Onglet Tableau de bord / Configuration)

  • Imaginez-les comme des classeurs totalement indépendants (jusqu'à 10).
  • Sélectionnez la configuration active avec le menu déroulant Configuration : dans le Tableau de Bord.
  • Chaque configuration a ses propres dossiers Backup/PJ et sa propre liste de profils.
  • Exemples : "Projets Investissements", "Suivi Qualité Usine", "Maintenance Bâtiment", "Projets Perso".
  • Vous pouvez nommer chaque configuration dans l'onglet Configuration pour plus de clarté.
  • Important : Les sauvegardes créées sont stockées dans le dossier Backup de la configuration active et sont spécifiques à celle-ci (leur nom de fichier inclut l'identifiant de la configuration, ex: C1_...).

2. Profils (Onglet Tableau de bord)

  • Au sein de la configuration active, vous pouvez créer plusieurs profils.
  • Sélectionnez le profil actif avec le menu déroulant Profil :.
  • Chaque profil contient son propre ensemble de Projets, d'Actions, et ses propres fichiers de sauvegarde, complètement isolé des autres profils (leur nom de fichier inclut aussi l'identifiant du profil, ex: ..._profilId_...).
  • Exemples : "ChefProjet_Dupont", "Fournisseur_XYZ", "GrandProjet_Alpha", "Utilisateur_Martin".
  • Utilisez les boutons Créer Profil, Supprimer Profil et Renommer (à côté du select Profil) pour les gérer.

➡️ Pensez d'abord à la Configuration (le grand contexte), puis au Profil (le sous-ensemble spécifique) que vous voulez utiliser.

Le Tableau de Bord : Votre Vue d'Ensemble

L'onglet Tableau de bord est votre point de départ et de synthèse.

Éléments Clés :

  • Contrôles Principaux :
    • Bouton 🔒 Autoriser dossiers (vu à l'étape 2).
    • Menus déroulants Configuration : et Profil : pour choisir votre contexte de travail.
    • Bouton Sauvegarder : Crée une sauvegarde manuelle immédiate du profil actif.
    • Boutons Créer Profil / Supprimer Profil / Renommer : Gèrent les profils de la configuration active.
  • Chargement de Sauvegarde :
    • Menu déroulant Sauvegarde : : Liste les sauvegardes (.json) existantes pour le profil actif (la plus récente en premier).
      ➡️ Note : Ce menu affiche uniquement les fichiers de sauvegarde qui correspondent à la Configuration ET au Profil actuellement sélectionnés.
    • Bouton Charger : Restaure l'état de l'application (projets, actions, journal...) à partir de la sauvegarde sélectionnée. IMPORTANT : Après avoir changé de Config ou de Profil, ce bouton clignotera, vous invitant à cliquer dessus pour activer le nouveau contexte.
  • Indicateurs Clés (Stats) : Nombre de projets actifs, actions en cours, en retard, terminées.
  • Statistiques par Projet : Un tableau récapitulatif du nombre d'actions par statut pour chaque projet du profil actif.
  • Journal d'Activité :
    • Affiche l'historique récent des créations, modifications et suppressions de projets et d'actions pour le profil actif.
    • Indique la date/heure, l'utilisateur (défini dans l'onglet Configuration) et les détails du changement.
    • Permet de choisir le nombre d'entrées à afficher (ex: 50 dernières, toutes...).
    • Possibilité de Vider le journal (action irréversible pour ce profil).
    • Cliquez sur les détails d'une modification pour potentiellement voir plus d'informations si le texte est long.

➡️ Utilisez cet onglet pour avoir une vue rapide de l'état d'avancement et pour naviguer entre vos différents contextes de travail.

Créer votre Premier Projet

Les actions doivent être rattachées à un projet. Commençons par en créer un.

  1. Assurez-vous d'être dans la bonne Configuration et le bon Profil (via le Tableau de bord).
  2. Allez dans l'onglet Projets.
  3. Cliquez sur le bouton ➕ Ajouter un projet. Un formulaire apparaît.
  4. Remplissez les champs :
    • Nom du projet : Nom descriptif (ex: "Amélioration Ligne Prod 3").
    • Préfixe : 2 à 4 lettres/chiffres pour l'identifiant unique (ex: ALP3, INV24).
    • Client/Fournisseur : Entité liée au projet.
    • Dates : Début et fin prévues.
    • Statut : État actuel du projet (Actif, Terminé...).
    • Description : Détails supplémentaires.
  5. Cliquez sur Enregistrer.

Le projet apparaît maintenant dans la liste. Chaque projet aura un numéro unique basé sur le préfixe et un compteur (ex: ALP3-001).

Cette création sera enregistrée dans le Journal d'Activité du Tableau de bord.

💡 Vous pouvez Modifier (✏️) ou Supprimer (🗑️) un projet via les icônes dans la liste.

Ajouter des Actions à un Projet

Maintenant que vous avez un projet, vous pouvez y ajouter des actions spécifiques.

  1. Allez dans l'onglet Actions.
  2. Cliquez sur le bouton ➕ Ajouter une action. Le formulaire s'affiche.
  3. Remplissez les champs :
    • Projet associé : Sélectionnez le projet créé précédemment dans la liste déroulante.
    • Description : Décrivez clairement l'action à réaliser.
    • Thématique : Catégorie (optionnel, ex: Mécanique, Elec, Software, Doc...).
    • Responsable : Qui doit réaliser l'action.
    • Dates : Début et limite.
    • Priorité : Haute, Moyenne, Basse.
    • Statut : En attente, En cours, Terminé, Bloqué.
    • Notes : Informations additionnelles, historique...
  4. Pièces Jointes (PJ) :
    • Cliquez sur Parcourir... (ou zone similaire) pour choisir un fichier.
    • Cliquez sur Ajouter PJ. Le fichier apparaît dans une liste temporaire.
    • Vous pouvez ajouter jusqu'à 5 fichiers par action.
  5. Cliquez sur Enregistrer l'action.

L'action apparaît dans le tableau. Elle aura un numéro unique lié au projet (ex: ALP3-001-001).

L'ajout de cette action sera également visible dans le Journal d'Activité.

💡 Les PJs sont physiquement copiées dans le dossier "PJ" de la configuration active lorsque vous enregistrez l'action.

Gérer les Actions et Pièces Jointes

L'onglet Actions est l'endroit où vous passerez le plus de temps.

Fonctionnalités Clés :

  • Liste des Actions : Tableau principal affichant toutes les actions (filtrées).
  • Filtrage :
    • Par Statut (cases à cocher).
    • Par Projet (menu déroulant).
    • Par texte dans chaque colonne (champs sous les en-têtes).
  • Modification (✏️) / Suppression (🗑️) : Via les icônes à droite de chaque ligne. Ces opérations sont tracées dans le Journal d'Activité.
  • Description Détaillée :
    • La colonne "Description" n'affiche qu'un aperçu.
    • Passez la souris sur la description pour voir le texte complet dans une info-bulle.
    • Cliquez sur la description pour l'éditer dans une fenêtre dédiée (plus pratique pour les textes longs).
  • Pièces Jointes (PJ) :
    • Si une action a des PJs, un bouton PJ (x) apparaît.
    • Cliquez dessus pour ouvrir une galerie d'aperçus (images, PDF si possible).
    • Dans la galerie, cliquez sur une miniature pour voir un aperçu plus grand ou sur "Télécharger".
    • Dans le formulaire de modification d'action, vous pouvez supprimer des PJs existantes (🗑️) ou en ajouter de nouvelles.
  • Export : Le bouton Exporter (PJ + CSV) crée un dossier contenant un fichier CSV des actions visibles et un sous-dossier avec les PJs correspondantes. Très utile pour partager un état d'avancement !

L'Onglet Configuration : Paramétrage Avancé

Cet onglet regroupe les paramètres et les fonctions de maintenance.

Sections Principales :

  • Gestion des Configurations (1-10) :
    • Permet de Nommer chaque configuration pour la reconnaître facilement (ex: "Projets R&D", "Support Client").
    • Boutons Redéfinir pour (re)définir les dossiers Backup et PJ pour chaque configuration individuelle. Les chemins actuels sont affichés.
  • Sauvegarder / Restaurer (Profil Actif) :
    • Sauvegarder Profil Actif : Crée une sauvegarde manuelle du profil actuellement sélectionné dans la configuration active.
    • Restaurer (depuis fichier...) : Permet de charger des données (projets, actions) à partir d'un fichier .json externe (attention, cela écrase les données actuelles du profil !).
  • Fonctions avancées Profils : (Non implémentées pour l'instant) Export/Import/Réparation groupés de profils.
  • Maintenance Numérotation :
    • Recalculer Numéros Actions : Réorganise les numéros séquentiels des actions (le dernier chiffre après le projet, ex: 001, 002...) pour qu'ils soient consécutifs au sein de chaque projet. Utile après des suppressions.
    • Réinitialiser Compteur Projets : (⚠️ Dangereux) Change le numéro qui sera attribué au PROCHAIN projet créé. À n'utiliser qu'en cas de problème majeur.

➡️ C'est ici que vous personnalisez l'application et gérez les aspects techniques.

Travailler à Plusieurs : Le Verrouillage

Si vous partagez les dossiers (Backup et PJ) sur un lecteur réseau pour collaborer, le système de verrouillage entre en jeu.

  • Objectif : Éviter que deux personnes modifient les mêmes données (le même profil dans la même configuration) en même temps, ce qui pourrait entraîner des pertes d'informations.
  • Fonctionnement : Quand vous chargez un profil (via le bouton Charger du Tableau de Bord), l'application tente de créer un petit fichier invisible (`.lock`) dans le dossier Backup de la configuration active. Ce fichier contient votre nom d'utilisateur et l'heure.
  • Si vous obtenez le verrou :
    • La bannière supérieure indique "Vous éditez: Profil [...]".
    • Vous pouvez modifier les projets et les actions normalement.
    • Le verrou est automatiquement rafraîchi tant que vous utilisez l'application.
  • Si le profil est déjà verrouillé par quelqu'un d'autre :
    • Un bandeau orange/jaune apparaît en haut, indiquant qui a verrouillé et depuis quand.
    • La bannière supérieure indique "Consultation: Profil [...] - Verrouillé.".
    • L'application passe en mode lecture seule : vous pouvez consulter, filtrer, exporter, mais pas ajouter, modifier ou supprimer de projets/actions. L'onglet Configuration reste utilisable.
    • Vous pouvez cependant consulter le Journal d'Activité pour voir les dernières modifications effectuées par l'autre utilisateur.
  • Expiration & Forçage : Si un verrou date de plus de 10 minutes, l'application considère qu'il est peut-être "périmé" (l'autre utilisateur a peut-être fermé son navigateur sans déverrouiller). Elle vous proposera alors de forcer la prise en main. Utilisez cette option avec prudence !

➡️ Ce système permet une collaboration basique en évitant les conflits directs sur les mêmes données.

Trucs & Astuces et Conclusion

Quelques Conseils :

  • Nommez bien vos Configurations et Profils : Cela rend la navigation beaucoup plus simple. Utilisez la fonction "Nommer" dans l'onglet Configuration.
  • Sauvegardes Régulières : Même si l'auto-sauvegarde existe, n'hésitez pas à faire une sauvegarde manuelle (via le Tableau de Bord ou Config) avant une opération importante ou à la fin de votre session.
  • Utilisez les Filtres : Le tableau des actions peut vite devenir long. Les filtres par statut, projet et texte sont vos meilleurs alliés pour retrouver une information.
  • Export pour Partage : L'export CSV+PJ est parfait pour envoyer un point d'avancement clair à des collègues ou des externes qui n'utilisent pas l'application.
  • Description Longue : Cliquez sur la description dans le tableau des actions pour l'éditer confortablement dans une fenêtre dédiée.
  • Dossiers Réseau : Pour le partage, placez les dossiers Backup et PJ sur un lecteur réseau accessible par tous les utilisateurs concernés. Assurez-vous que tous utilisent la même lettre de lecteur ou chemin UNC si possible (bien que l'application utilise des références internes une fois le dossier choisi).

Conclusion

Vous avez maintenant une bonne vue d'ensemble des fonctionnalités de l'application Suivi de Projets et Actions. Son potentiel réside dans sa capacité à s'adapter à différents contextes grâce aux Configurations et Profils.

N'hésitez pas à explorer et à l'adapter à votre propre méthode de travail !
